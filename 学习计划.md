# Vue 3 零基础学习计划

## 背景

目标用户是完全零基础的小白（没接触过 Vue，甚至可能没接触过前端框架），需要用大白话、中文注释、完整可运行代码，从 0 到 1 掌握 Vue 3。每讲一个知识点都要：**先解释概念 → 再写代码 → 再讲用法**。不跳步、不用黑话、不省略步骤。

所有文件使用**相对路径**，放在项目当前目录下。

---

## 学习路线总览（共 7 个阶段）

### 第一阶段：环境搭建 + 第一个 Vue 应用
**目标**：把电脑变成能运行 Vue 的环境，跑起来第一个页面

内容：
1. 安装 Node.js（解释：Node.js 是什么，为什么装它）
2. 创建 Vue 3 项目（使用 `npm create vue@latest`）
3. 项目文件结构介绍（每个文件夹/文件是干啥的）
4. 启动开发服务器，在浏览器看到页面
5. 修改页面内容，感受"热更新"

**产出**：浏览器成功显示"Hello Vue 3！"

---

### 第二阶段：模板语法 —— 在 HTML 里写 JS
**目标**：学会在 `<template>` 里显示数据、写表达式

知识点：
1. **插值表达式 `{{ }}`** —— 把 JS 数据显示到页面上
2. **`v-text` 和 `v-html`** —— 纯文本 vs HTML 字符串
3. **`v-bind` （简写 `:`）** —— 动态绑定 HTML 属性（如 src、href、class）
4. **`v-on` （简写 `@`）** —— 绑定点击、输入等事件
5. **`v-model`** —— 表单输入双向绑定

每节课：一个 `.vue` 文件，完整可运行，中文注释。

---

### 第三阶段：响应式数据 —— Vue 的核心魔法
**目标**：理解"数据变了，页面自动变"的原理

知识点：
1. **`ref()`** —— 定义单个响应式数据
2. **`reactive()`** —— 定义对象/数组响应式数据
3. **`computed()`** —— 计算属性（依赖别的数据自动算出新值）
4. **`watch()`** —— 侦听器（数据变化时做事情）
5. **`v-if` / `v-show`** —— 条件渲染
6. **`v-for`** —— 列表渲染

每节课：一个 `.vue` 文件 + 讲解注释 + 实际演示效果。

---

### 第四阶段：组件化开发 —— 搭积木
**目标**：学会把页面拆成多个小组件，组合成大页面

知识点：
1. **组件概念** —— 什么是组件，为什么用组件
2. **父传子 `defineProps`** —— 父组件给子组件传数据
3. **子传父 `defineEmits`** —— 子组件通知父组件
4. **插槽 `<slot>`** —— 在组件里留"坑位"放内容
5. **组件之间的分工** —— 容器组件 vs 展示组件

每节课：父子组件两个文件，完整可运行。

---

### 第五阶段：Vue Router —— 多页面跳转
**目标**：学会做多页面应用，点链接切换页面

知识点：
1. **路由是什么** —— 类比：不同 URL 对应不同页面
2. 安装和配置路由 —— `createRouter`
3. **`<router-link>` 和 `<router-view>`** —— 导航链接 + 页面出口
4. **动态路由 `:id`** —— 详情页参数
5. **编程式导航** —— JS 代码跳转页面

---

### 第六阶段：Pinia 状态管理 —— 全局数据共享
**目标**：解决"多个组件共享同一份数据"的问题

知识点：
1. **什么是状态管理** —— 为什么需要 Pinia
2. **定义 Store** —— `defineStore`
3. **在组件中使用 Store** —— 读取和修改数据
4. **Action（动作）** —— 封装修改逻辑
5. **Getter（计算属性）** —— 从 store 衍生出新数据

---

### 第七阶段：综合实战 —— 待办事项 App
**目标**：把所有知识串起来，做一个完整的小应用

功能清单：
1. 添加待办事项
2. 标记完成/未完成
3. 删除待办
4. 筛选（全部/已完成/未完成）
5. 路由区分页面（首页 + 关于页）
6. 使用 Pinia 存储数据

涉及知识点：
- 组件化（TodoList、TodoItem、AddTodo）
- 响应式数据（ref、computed）
- 事件处理（@click、@submit）
- 双向绑定（v-model）
- 条件/列表渲染（v-if、v-for）
- 路由（router-link、router-view）
- 状态管理（Pinia store）
- Props / Emits

---

## 教学规范（每节课必须遵守）

### 代码文件格式
```vue
<!-- 第X课：知识点名称 -->
<!-- 作者：讲师 -->
<!-- 日期：YYYY-MM-DD -->

<template>
  <!-- 页面结构写在这里 -->
</template>

<script setup lang="ts">
// JS/TS 逻辑写在这里
// 每行代码都要有中文注释
</script>

<style scoped>
/* 样式写在这里，scoped 表示样式只对当前组件生效 */
</style>
```

### 每节课的结构
1. **概念解释**（大白话，不打比方、不扯远的）
2. **完整代码**（复制就能跑的 `.vue` 文件）
3. **逐行讲解**（代码里每行都有中文注释）
4. **动手修改**（"你试着把 X 改成 Y，看看效果"）
5. **小结**（今天学了什么，用一句话总结）

### 文件命名规则
```
lesson-01-hello-world/
├── App.vue          # 主入口，学生直接看这里
├── README.md        # 本课讲解文字

lesson-02-template/
├── App.vue
├── README.md

...以此类推
```

### 路径规范
所有文件都使用**相对路径**，相对于项目根目录 `./` 创建，绝不使用绝对路径。
