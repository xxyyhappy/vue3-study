> **学习日期**：2026-05-10

---

## 01. Node.js —— 让 JS 能在电脑上运行

### 大白话

以前 JavaScript 只能在浏览器里"活着"，有了 Node.js，你的电脑自己就能执行 JS 代码了。

就像**鱼和水**的关系：
- 浏览器 = 鱼缸（JS 只能在鱼缸里游）
- Node.js = 把鱼放到大海里（JS 可以在电脑上自由跑了）

**我们为什么需要 Node.js？** 因为 Vite（我们的开发服务器）就是用 Node.js 跑的。

---

## 02. Vite —— 开发服务器 + 打包工具

### 大白话

Vite 就像**一个智能厨房**：

- **开发时**：你切菜（写代码），厨房实时帮你炒好（热更新预览）
- **打包时**：把所有菜打包成便当（压缩优化），交给用户吃（上线部署）

| 模式 | 作用 | 类比 |
|------|------|------|
| 开发模式（`npm run dev`） | 启动本地服务器，实时预览 | 厨房现场做菜 |
| 生产构建（`npm run build`） | 打包优化项目 | 做成速冻便当 |

---

## 03. .vue 文件 —— 单文件组件

### 大白话

一个 `.vue` 文件就是一个**独立的零件**，包含 3 个部分：

```
┌─────────────────────┐
│  <template>         │  ← 写 HTML，决定页面长什么样
│  结构                │
├─────────────────────┤
│  <script setup>     │  ← 写 JS，控制页面的行为逻辑
│  行为                │
├─────────────────────┤
│  <style scoped>     │  ← 写 CSS，给页面化妆打扮
│  样式                │
└─────────────────────┘
```

就像**一个人的三个要素**：
- 骨架（HTML）—— 长什么样
- 大脑（JS）—— 会做什么
- 衣服（CSS）—— 穿什么好看

```vue
<template>
  <!-- HTML 写这里 -->
  <p>Hello, Vue!</p>
</template>

<script setup>
// JS 逻辑写这里
const msg = 'Hello'
</script>

<style scoped>
/* CSS 写这里，scoped 让样式只对当前组件生效 */
p { color: red; }
</style>
```

## 04. 热更新（HMR）—— 改完代码，浏览器自动刷新

### 大白话

热更新就像**你在画画时，旁边有个助手随时同步你的每一笔**：

- 你画了一笔（改了代码）
- 助手立刻同步到展示板上（浏览器自动更新）
- 你不用每次都重新画整张画（不用手动刷新浏览器）

**开发体验**：改代码 → `Ctrl + S` 保存 → 浏览器瞬间更新。

## 05. 项目文件结构

```
first-vue-app/
├── index.html           # 浏览器加载的入口 HTML（页面外壳）
├── package.json         # 项目"身份证"（名称、依赖、命令）
├── vite.config.js       # Vite 的配置文件
└── src/
    ├── main.js          # Vue 应用的入口 JS
    └── App.vue          # 根组件（主要写代码的地方）
```

### 每个文件的作用

| 文件 | 大白话解释 |
|------|-----------|
| `index.html` | 浏览器打开的页面外壳，里面有 `<div id="app">` 等着 Vue 往里填内容 |
| `package.json` | 记录项目叫什么、用了哪些依赖、有哪些可用命令 |
| `vite.config.js` | 告诉 Vite 怎么运行（比如处理 `.vue` 文件需要什么插件） |
| `src/main.js` | Vue 的起点：创建 Vue 应用，挂载到 HTML |
| `src/App.vue` | 根组件，整个页面从这里展开 |

---

## 06. main.js 逐行解读

```js
import { createApp } from 'vue'  // 从 vue 框架中导入 createApp 函数
import App from './App.vue'      // 导入根组件 App.vue

// createApp(App)  → 创建 Vue 应用
// .mount('#app')  → 挂载到 index.html 的 <div id="app"> 上
createApp(App).mount('#app')
```

**运行流程**：

```
index.html 中的 <div id="app"></div>  （一个空的容器）
                    ↑
            main.js 把 App.vue 挂载上去  （往容器里塞内容）
                    ↑
            App.vue 开始渲染页面         （内容显示出来了）
```

## 07. 常用命令

| 命令 | 作用 | 大白话 |
|------|------|--------|
| `npm create vue@latest 项目名` | 创建新项目 | 买一套新房子的毛坯 |
| `npm install` | 安装依赖 | 把家具搬进来 |
| `npm run dev` | 启动开发服务器 | 开始装修，边装边看效果 |
| `npm run build` | 打包构建 | 装修完拍照，给客户看成品 |

## 08. 可能遇到的问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `npm install` 很慢 | 网络问题，默认从国外下载 | 用淘宝镜像 `npm config set registry https://registry.npmmirror.com` |
| 端口被占用 | 上次的服务器没关 | Vite 会自动换端口，看终端提示的地址 |
| 浏览器白屏 | 没等服务器启动 | 看终端是否显示 `ready in xx ms` |
| 报错找不到文件 | 配置文件里引用了不存在的文件 | 检查路径是否正确 |
